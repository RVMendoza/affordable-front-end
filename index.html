Why is templating with Jekyll amazing?

  • Backend-y things, without the back end
  • You don't have to hand code references to you css and js every page
  • You can create modules and call them (headers, footers)
  • Simple blogging
  • Markdown
  • Variables let you be lazy.

Set up Jekyll in Three Steps

Install Jekyll, make a new page, and serve it

  1. gem install jekyll
  2. jekyll new myblog
  3. jekyll serve --watch
    • Basically a compass watch (http://localhost:4000)
    • It compiles your site, and then you push it to git hub
